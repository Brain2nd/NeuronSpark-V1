## §1 设计背景与决策

### 1.1 起点：v3 现状 (supra-threshold release)
- `output = relu(V_pre - v_th)`，软重置 `V_post = min(V_pre, v_th)`
- 反向梯度精确 ReLU：`∂output/∂V_pre = 𝟙[V_pre>v_th]`，无 surrogate
- 问题：发放时膜被钉回 v_th，**「累积了多强的上下文」从膜里被抹掉**，只活在 output 流里

### 1.2 设计讨论（与作者）
- 用户原则：「**历史膜电位承载历史上下文信息**，硬重置不可讨论」
- 候选：(A) supra 现状 / (B) quantal 量子化 (output=v_th·s, 余量留膜里) / (C) 硬重置（OFF the table）/ (D) 分数释放
- 用户决策：**改为 quantal**，AHP（后超极化 / 不应期）一同实现
- 用户初始问：「AHP 会不会引起梯度消失？」
- 我（错误地）笃定答：「不会，supra 下 `∂V_post/∂V_pre=1-s_hard` 不变」—— 但只在 supra 验证了，没在 quantal 重做数学；后果见 §3.4

### 1.3 量子化释放正式定义
```
spike      = 𝟙[V_pre > v_th]                    (硬指示子, forward 用)
output     = v_th · spike                        (固定一份 v_th, 矩阵-向量层面)
V_post     = V_pre - v_th·s - ahp·s              (软重置: 余量 V_pre-v_th 留膜; AHP 额外下压)
反向 (surrogate, sigmoid'(α·x))：
  ∂out/∂V_pre   = v_th · α·σ·(1-σ) ≈ v_th·α/4 ≈ 0.02   ← **比 supra 弱 ~50x**
  ∂V_post/∂V_pre = 1 - (v_th + ahp) · α·σ·(1-σ)
```
这个 ~50x 弱梯度是后续 quantal 训练困难的核心源头。

### §3.2 实验 2: 本地 v_th 缩放探针（D=384 小模型）
**目的**：探查「初始化要不要给 quantal 单独缩放 v_th」。

**配置**：D=384, N=8, K=8, 8 层；D_ff=768；7 个配置；30 步 short overfit + 前后发放率。

| 配置 | hidden μ init | hidden μ @30步 | loss 30步 |
|---|---|---|---|
| supra 基准 | 0.40 | 0.19 | 11.79→9.14 |
| **quantal 原样** | 0.22 | **0.32（回升）** | 11.75→**9.08** |
| quantal v_th×5 vmin0.1 | 0.24 | 0.25 | 11.78→9.13 |
| quantal v_th×10 vmin0.2 | 0.22 | 0.24 (gate 掉到 0.07) | 11.82→8.90 |
| quantal v_th×10 vmin0.3 | 0.21 | 0.23 | 11.82→8.91 |
| **quantal v_th×20 vmin0.5** | 0.21 | **NaN @ step1** | — |

**结论**：v_th 缩放 ×5 ~ ×10 不解决问题，反而抑制 gate（FFN gate 的 v_th 也被一并缩大）；×20 直接 NaN。**小模型里 quantal 原样跟 supra 打平**。

→ 「怎么初始化」直接结论：**保持现有初始化，不给 quantal 单独缩放 v_th**。

但 §3.1 大模型 quantal hidden 是 0.006，跟小模型的 0.22 不一致 —— **模型尺寸效应**。详见 §4。

### §3.3 实验 3: Muon+Adam+Lion (MAL) ablation
**目的**：测试 Lion 是否能让稀疏 SNN 梯度（quantal 50x 弱 + 时间稀疏）下 SNN 学起来。

**动机**：Adam baseline (§3.1) hidden 全塌。假设是优化器问题 —— quantal 的 spike 梯度 `∂out/∂V_pre = v_th·surrogate' ≈ 0.02`（比 supra 弱 50x）+ 时间稀疏（rare spike → 大多数 batch 零梯度）= 「小幅 + 稀疏 + 带噪」最不利情景。Lion (Chen et al. 2023) 的 `update = lr·sign(β1·m + (1-β1)·g)` 是 scale-invariant（无视梯度幅度）+ momentum 平滑（对稀疏 nonzero 事件鲁棒），理论上更适合。

**实现**：commit `a6e3376` 的 `MoonshotMuonAdamLion`（`utils/muon_adam_lion.py`）：
- Muon (Moonshot abs-RMS scaling) 给 matrices（W_in/W_out/W_th_x/ffn_*_proj/...）
- Adam 给 embedding + RMSNorm gains
- **Lion 给神经元标量/向量参数**（commit `57b0783` 后拆成两组：`neuron-pos` = v_th + ahp 带 `clamp_(min=1e-4)`；`neuron-free` = w + b_beta/b_alpha/b_th 无 clamp）
- DeepSpeed 兼容：ZeRO-0 round-robin + all_gather 给 Muon 矩阵（继承自原 MoonshotMuonWithAuxAdam）；Lion/Adam per-rank-replicated

**预测试（8 步 smoke，quantal+AHP, D=512 12 层模型, 4090）—— 决定要不要上 MAL**:
| | hidden μ step 0 | hidden μ step 7 |
|---|---|---|
| Adam (lr=2e-3) | 0.006 | **0.000（塌）** |
| Muon+Adam+Lion (默认 LR) | 0.006 | **0.037（活, 6x init, 上升）** |
→ MAL 8 步内就把 hidden 从死循环里拉出来，决定上 1500 步全量。

**4090 1500 步 run（默认 LR：muon_lr=0.02, adam_base_lr=2e-4, lion_lr=5e-4×mult10 = 5e-3）**:
- **supra**: step 600 loss 3.97, hidden μ0.196；step 1000 loss 5.30, hidden μ0.149；step 1200 loss 5.68, hidden μ0.140 —— **健康，未完成（截至记录）**
- **supra+AHP**: step 600 loss 4.64, hidden μ0.191；step 1000 loss 5.51, hidden μ0.161；step 1200 loss 5.84, hidden μ0.158 —— **健康**
- **quantal**: NaN @ step 94
- **quantal+AHP**: NaN @ step 71

**关键观察**：
1. **Lion 把 supra 的 SNN 救活了** —— hidden 不再随训练塌掉。Adam baseline 同模型 step 400-600 hidden=0.000（已死）；MAL step 600 hidden μ0.196 健康。supra MAL 跑到 step 600 loss 已 3.97，比 Adam baseline 整个 1500 步的 7.33 低 ~3.4 nat（对比要等 MAL supra 完整跑完）
2. **quantal 在 MAL 下激活了 hidden（μ0.006→0.131 over 200 步）但很快 NaN** —— 见 §3.4 调查

## §4a quantal 保住 K 步动力学（重要观察 — 副带解决「PonderNet 退化 K」）

实测对比（MAL 优化器, 1500 步）：
- **supra**: E[K]_out 从 6.5 (init) → step 1000 **3.35** → step 1400 **2.60**（PonderNet 学到只跑 ~2-3 帧）
- **quantal / quantal+AHP**: E[K]_out 一直稳在 **~7.6-8.1**（PonderNet 学到值得跑 ~8 帧）

**机制**：
- supra 的 `output = relu(V_pre - v_th)` + 软重置到 v_th → 发放就把超阈部分掏空、膜钉回阈值 → **膜几帧就 settle（收敛到稳态）** → 多跑帧没额外信息 → PonderNet 学着提前 halt → **K 退化到 ~1，K 步串行 SNN 动力学被实质废掉**（架构核心卖点失效）
- quantal 的 `V_post = V_pre - v_th·s - ahp·s`，**超阈余量留膜里** → 膜跨帧持续积分、攒到够整数份 v_th 就 burst（rate code）→ **每多跑一帧都是真计算** → PonderNet 学到「这值得跑更多帧」→ E[K] 稳在高位 → **K 步递归动力学保住**

→ quantal 一个改动同时：① 膜保留历史上下文（原始设计动机，§1）② 让多步递归「load-bearing」从而 PonderNet 不会把 K 退化掉（本来要单独解决的问题）。**这是 quantal vs supra 的一个独立优势，且不依赖小训练规模**（机制是结构性的）。

注：副作用 —— quantal 推理时 early-stop 省的 FLOP 更少（E[K]≈8 vs supra E[K]≈2.6）。但 supra 的 E[K]≈2.6 是「把 SNN 跑废了」换来的，不算真省。trade-off：quantal 保留 SNN 计算能力但推理慢一些。

## §4 模型尺寸效应（重要观察）
小模型（D=384 / 8 层）quantal 原样训练正常（hidden 0.22→0.32）；大模型（D=512 / 12 层）quantal hidden 起步就 0.006、塌到 0。

**原因（理论）**：quantal output = v_th·s ≈ 0.02; W_in (D=512) → DN=8192, kaiming `~N(0, 2/D)`. (W_in @ output) per-channel std = sqrt(2·E[output²]) = sqrt(2·0.02²·0.49) ≈ 0.02. 即 hidden 神经元的 V_pre 期望幅度 ~0.02，hidden v_th ≈ v_th_min + |...| ≈ 0.02-0.05。**V_pre ~ v_th 量级 → 发放率随机** → 略低于阈值时几乎不发放。

更深的层级联衰减：input_neuron 输出 ~0.02 → W_in → hidden V_pre ~0.02 → hidden 输出 ~0.02·firing_rate ≈ 0.006 → 下一层 W_in → ~0.006 → ...

→ quantal 在大模型下需要**初始化解决级联问题**（论文核心 contribution 之一）。

## §4b 记忆归因：历史信息在 SNNBlock vs SNNAttention 之间如何分配（`scripts/v4_memory_analysis.py`）

**目标**：数学严谨地验证作者的推测 —— SNNAttention（类海马体线性脉冲注意力）= 长程记忆；SNNBlock（SelectivePLIF SSM-state 类比）= 短/近期记忆；SNNBlock 内部本身也有长短期分化。

### E1 β 时间常数谱（design-level，定义：膜递推 ∂V_t/∂V_{t-1}=β → 影响按 β^Δ 衰减 → 记忆时间常数 τ = -1/ln β）
在 290M 配置模型上（fresh-init，因为 β 的 base 来自初始化设计）：

| 组件 | β 来源 | τ 范围（单位 = K-frame 步）| 桶分布 [<3, 3-10, 10-30, 30-100, >100] | 记忆性质 |
|---|---|---|---|---|
| input_neuron1/2, gate_neuron, up_neuron, output_neuron (PLIFNode) | sigmoid(w), w~N(0,0.5²) | τ ≈ 0.7-3.9（中位 ~1.4）| ~97% <3 | **快 / 近期专属**（~1-2 步）|
| SNNBlock hidden_neuron (SelectivePLIF) | β_base=sigmoid(b_beta), b_beta init = logit linspace(0.80,0.99) + N(0,0.1) 扰动 | τ_base ≈ 4-110（中位 ~9）| ~0% <3, ~55% [3-10], ~31% [10-30], ~11% [30-100], ~3% >100 | **多时间尺度**（4 到 110）—— **SNNBlock 内部就有长短期分化** |

SNNAttention：线性注意力 + RoPE → 位置 s 对位置 t 输出的有效权重 ∝ Σ_j cos((t-s)·θ_j)，θ_j = base^(-2j/d_head)，base=10000, d_head=128（=D_key）。「记忆波长」λ_j = 2π/θ_j：

| | 最短 λ（最高频）| 中位（25/50/75%）| 最长 λ（最低频）|
|---|---|---|---|
| SNNAttention RoPE | 6.3 tokens | 60.7 / 586 / 5652 | **54410 tokens** |

→ **设计层面结论（支持作者推测）**：
- PLIFNode 神经元：~1-2 步 → 纯近期
- SNNBlock hidden_neuron：~4-110 K-frame 步（≈ token，因 carry 是 per-token）→ 短到中程，**内部多时间尺度**
- SNNAttention：~6 到 ~54000 tokens → 短到**极长程**
- **SNNBlock 负责短/近期，SNNAttention 是长程记忆的载体**；且 SNNBlock 自身有长短期分化（b_beta linspace [0.80,0.99]）。

### E2 经验状态自相关（定义：ρ(Δ) = mean_c pearson_t(state_t[c], state_{t-Δ}[c])；衰减到 1/e 的 Δ = 经验记忆时间常数）
在 fresh-init 模型上、val 序列（seq 256）—— 抓 SNNBlock hidden_neuron 跨 token 携带的膜电位 V_post[k_t]：
- 各层 ρ(Δ): Δ1≈0.52-0.63, Δ2≈0.30-0.39, Δ4≈0.12-0.19, Δ8≈0.03-0.10, Δ16≈0-0.06, Δ≥32≈0
- **经验 τ_emp(1/e) ≈ 1.7-2.3 tokens**

→ 经验上 SNNBlock 携带状态只自相关 ~2 tokens —— 远小于 β_base 的中位 ~9（更别说 ~110 的尾巴）。原因：(a) PLIFNode 们 β≈0.5 → τ≈1.4 → 这些「快」神经元主导了 input_neuron 链；(b) fresh-init 时膜被当前 token 的新输入主导，递归历史项小（β^K 内衰减）；(c) 只有 hidden_neuron 的 ~3% 慢通道（β≈0.99）携带 ~100-token 记忆，混入均值后被快通道淹没。**训练后再测会更有意义**（模型可能学会用慢通道）。

### E3 因果扰动影响（定义：扰动 emb at pos T-1-Δ → 末位 logits 变化 ‖Δlogits_T‖/‖ε‖ = influence(Δ)；component 归因：关掉 SNNAttention 跨位置混合后 influence(Δ) 在长 Δ 塌掉 ⇒ 长程靠 SNNAttention）
- **当前状态：有 bug，结果不可信** —— fresh-init 上 influence(Δ) 全是噪声幅度（~20）无衰减趋势，no-attn-mix 的 ratio 也接近 1。怀疑全局 `torch.cumsum` monkeypatch 在大模型（有 `checkpoint` 包裹）下没可靠传到 SNNAttention 层。待：① 在 trained ckpt 上复测；② 换更直接的 ablation（直接 override `SNNAttentionDecoderLayer.forward_parallel` 跳过 attn 子层）。
- **架构论证（不依赖实验，结构性事实）**：模型里跨位置信息路径只有三条：(1) SNNAttention 的无衰减 cumsum（RoPE 调制）—— 跨度 6 到 ~54000 tokens；(2) SNNBlock 的 segmented-PLIF 跨 token carry —— 快通道 ~1-2 tokens，慢通道（~3%）~100 tokens；(3) SNNDecoderLayer 的皮层侧向连接 —— 1 token 回看。**所以任何超过 ~100 tokens 的依赖必然走 SNNAttention**；~10-100 tokens 范围 SNNBlock 的慢通道和 SNNAttention 都能做；~1-10 tokens 范围三者都能。

## §5 LSUV 初始化本地测试（commit `0a50df8` scaffold）
**测试**（D=128, N=4, 4 层小模型, quantal+AHP）:
- LSUV 前: hidden μ=0.005, input μ=0.429
- LSUV 后（v_th = (1-0.4) 分位, n_passes=2）: hidden μ=**0.000**（SelectivePLIF hidden 没被 LSUV 碰，因为它的 v_th 数据相关无静态参数）, input μ=0.475
- LSUV + Lion 20 步: hidden 0.000 → 0.006（慢）
- 对比：不 LSUV + Lion 8 步: hidden 0.006 → 0.037（更快！）
→ **当前 LSUV 方案反而拖慢** —— quantile-init 把 v_th 设小、output `v_th·s` 更小、级联衰减更重。
→ 待改：(a) `v_th = c·std(V_pre)` (c≈0.5) 最大化 output 幅度（不是按 firing rate 最小化 v_th）；(b) hidden_neuron 通过调 `v_th_min` 或 scale `W_in` 来处理（不是改静态 v_th）。
