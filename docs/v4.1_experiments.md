# V4.1 量子化释放 (quantal) + AHP 实验日志

**起始日期**: 2026-05-11
**分支**: `v4`
**架构基线**: V4 = v3 删 conv1d + token-level 残差流 + segmented PLIF（携带第 k_t 帧）+ output_neuron k_predictor + Triton fused kernels

本文记录 V4.1 神经元设计（量子化释放 spike form + 后超极化 AHP）从设计到实测的全过程，所有 commit hash / 日志路径 / 复现命令都列在文中，为后续写论文提供原始数据。

> **TL;DR（2026-05-12 当前状态）**：v4.1 = 把 v3 的 supra-threshold release 换成 quantal release（`out = v_th·s`，剩余余量留膜里携带历史）+ bias 重构（Llama 式无偏置）。**关键障碍是优化训练**：① Adam 下 SNN 神经元全死 → 换 MAL（Muon 矩阵 + Adam embed/norm + Lion 神经元参数）→ 神经元全活、loss 低 2-3 nat；② quantal 训练偶发 NaN → 折腾很久（β-cap / vth_reg / AHP / muon_wd 都不是根治）→ 真根因 = quantal `out=v_th·s` 让 loss 驱动 `v_th_hidden` 涨大 + Muon 推 W_th_x → 膜递归 backward Jacobian `J = β·(1−(v_th+a)·g_surr)` 在 `v_th·α/4 > 1` 时 `|J|>1` → BPTT 发散 → 修法 = adaptive surrogate sharpness `α_eff = min(α, 4/(v_th+a))` 保证 `J ∈ [0,β]` 恒 contractive。③ 初始化用 LSUV v2（校准 v_th 到发放率分位数 + scale W_in 让 hidden 发放 ~0.3），否则 quantal hidden 塌到 ~0.6%。**当前推荐配置**：quantal + MAL(muon_lr=0.005, muon_wd=0.01, 自带 stochastic rounding) + grad_clip=0.5 + LSUV v2 + adaptive-α + NaN-skip(兜底)；**全模型 bf16**（含神经元参数 .w/.v_th/.ahp —— SR 保证小更新累积），仅 k_predictor EMA buffer fp32。下一步：Track A 组件级记忆归因 / Track B Mamba 类实验 / Track C 全量预训练（靠后）。详见 §3.0 时间线。

## §1 设计背景与决策

### 1.1 起点：v3 现状 (supra-threshold release)
- `output = relu(V_pre - v_th)`，软重置 `V_post = min(V_pre, v_th)`
- 反向梯度精确 ReLU：`∂output/∂V_pre = 𝟙[V_pre>v_th]`，无 surrogate
- 问题：发放时膜被钉回 v_th，**「累积了多强的上下文」从膜里被抹掉**，只活在 output 流里

### 1.2 设计讨论（与作者）
- 用户原则：「**历史膜电位承载历史上下文信息**，硬重置不可讨论」
- 候选：(A) supra 现状 / (B) quantal 量子化 (output=v_th·s, 余量留膜里) / (C) 硬重置（OFF the table）/ (D) 分数释放
- 用户决策：**改为 quantal**，AHP（后超极化 / 不应期）一同实现
- 用户初始问：「AHP 会不会引起梯度消失？」
- 我（错误地）笃定答：「不会，supra 下 `∂V_post/∂V_pre=1-s_hard` 不变」—— 但只在 supra 验证了，没在 quantal 重做数学；后果见 §3.4

### 1.3 量子化释放正式定义
```
spike      = 𝟙[V_pre > v_th]                    (硬指示子, forward 用)
output     = v_th · spike                        (固定一份 v_th, 矩阵-向量层面)
V_post     = V_pre - v_th·s - ahp·s              (软重置: 余量 V_pre-v_th 留膜; AHP 额外下压)
反向 (surrogate, sigmoid'(α·x))：
  ∂out/∂V_pre   = v_th · α·σ·(1-σ) ≈ v_th·α/4 ≈ 0.02   ← **比 supra 弱 ~50x**
  ∂V_post/∂V_pre = 1 - (v_th + ahp) · α·σ·(1-σ)
```
这个 ~50x 弱梯度是后续 quantal 训练困难的核心源头。

**关键观察**：
1. **hidden 神经元 4 个配置全塌**（supra 起点 0.30 → step 400 起塌；quantal 从 step 0 就 ~0.006）
2. **quantal 把 FFN gate 也搞死了**（supra 0.47 vs quantal 0.00）
3. quantal loss 反而最低 —— 因为 SNN 块和 FFN gate 都死，模型退化成「线性残差 + attention」，简单结构在 1500 步小数据里训得快
4. **不能用此对比判断 quantal 优劣** —— 比的是僵尸的颜色

**根因分析**：
- supra hidden 塌：1500 步 / 单 shard / 100 万 token 太少，SNN 块的复杂多步递推动力学没被推起来；模型走 W_skip + attn + FFN 路径就够用了。**注意**: 不是「SNN 不重要」，是「这点训练量推不动这个复杂的 recurrent 神经元」
- quantal FFN 塌：quantal 输出 `v_th·s ≈ 0.02` 太小，downstream FFN 几乎收不到信号

### §3.3 实验 3: Muon+Adam+Lion (MAL) ablation
**目的**：测试 Lion 是否能让稀疏 SNN 梯度（quantal 50x 弱 + 时间稀疏）下 SNN 学起来。

**动机**：Adam baseline (§3.1) hidden 全塌。假设是优化器问题 —— quantal 的 spike 梯度 `∂out/∂V_pre = v_th·surrogate' ≈ 0.02`（比 supra 弱 50x）+ 时间稀疏（rare spike → 大多数 batch 零梯度）= 「小幅 + 稀疏 + 带噪」最不利情景。Lion (Chen et al. 2023) 的 `update = lr·sign(β1·m + (1-β1)·g)` 是 scale-invariant（无视梯度幅度）+ momentum 平滑（对稀疏 nonzero 事件鲁棒），理论上更适合。

**实现**：commit `a6e3376` 的 `MoonshotMuonAdamLion`（`utils/muon_adam_lion.py`）：
- Muon (Moonshot abs-RMS scaling) 给 matrices（W_in/W_out/W_th_x/ffn_*_proj/...）
- Adam 给 embedding + RMSNorm gains
- **Lion 给神经元标量/向量参数**（commit `57b0783` 后拆成两组：`neuron-pos` = v_th + ahp 带 `clamp_(min=1e-4)`；`neuron-free` = w + b_beta/b_alpha/b_th 无 clamp）
- DeepSpeed 兼容：ZeRO-0 round-robin + all_gather 给 Muon 矩阵（继承自原 MoonshotMuonWithAuxAdam）；Lion/Adam per-rank-replicated

**预测试（8 步 smoke，quantal+AHP, D=512 12 层模型, 4090）—— 决定要不要上 MAL**:
| | hidden μ step 0 | hidden μ step 7 |
|---|---|---|
| Adam (lr=2e-3) | 0.006 | **0.000（塌）** |
| Muon+Adam+Lion (默认 LR) | 0.006 | **0.037（活, 6x init, 上升）** |
→ MAL 8 步内就把 hidden 从死循环里拉出来，决定上 1500 步全量。

**4090 1500 步 run（默认 LR：muon_lr=0.02, adam_base_lr=2e-4, lion_lr=5e-4×mult10 = 5e-3）**:
- **supra**: step 600 loss 3.97, hidden μ0.196；step 1000 loss 5.30, hidden μ0.149；step 1200 loss 5.68, hidden μ0.140 —— **健康，未完成（截至记录）**
- **supra+AHP**: step 600 loss 4.64, hidden μ0.191；step 1000 loss 5.51, hidden μ0.161；step 1200 loss 5.84, hidden μ0.158 —— **健康**
- **quantal**: NaN @ step 94
- **quantal+AHP**: NaN @ step 71

**关键观察**：
1. **Lion 把 supra 的 SNN 救活了** —— hidden 不再随训练塌掉。Adam baseline 同模型 step 400-600 hidden=0.000（已死）；MAL step 600 hidden μ0.196 健康。supra MAL 跑到 step 600 loss 已 3.97，比 Adam baseline 整个 1500 步的 7.33 低 ~3.4 nat（对比要等 MAL supra 完整跑完）

**诊断结果（quantal-no-AHP, muon_lr=0.02, lion_lr=1e-4, NaN @ step 157）**:
- **NaN 是反向梯度溢出，不是前向爆炸** —— forward hooks 显示所有 layer 输出 finite，loss=5.41 finite。backward 触发第一个 NaN grad 在 `snn.embed_tokens.weight` (随后所有层 norm/v_th 的 grad 都 inf)
- **罪魁是 Muon 把 matrix 权重越推越大**：

  | 量 | step 0 | step 150 | 倍数 |
  |---|---|---|---|
  | W_in max-abs | 3.6e-2 | 3.83 | **100x** |
  | W_out max-abs | 6.1e-2 | 3.34 | **55x** |
  | W_th_x max-abs | ~0.4 | 5.22 | ~13x |
  | gnorm | 0.27 | 0.99（每步顶死 clip=1.0）| 撞顶 |
  | hidden_fr_μ | 0.006 | 0.12-0.13 | 救活了 |
  | v_th min | 0.0074 | 0.0001（撞 clamp_min）| 钳住了 |

- **机制定量**：Moonshot scaling = `0.2 · √max(rows,cols) = 0.2·√8192 ≈ 18.1`。每步 W_in 元素更新 RMS = `muon_lr · 18.1 = 0.02·18.1 = 0.36`。150 步 × momentum 平滑 → 元素从 ~0.04 长到 ~3.8。**完全符合 Moonshot 公式**。
- **为什么 supra 能用同 LR 不爆**：supra 输出 `relu(V_pre - v_th)` 随 V_pre 同步缩放 → 系统自平衡，W_in 长不会失控。quantal 输出锁死在 v_th（~0.02-0.25），W_in 大 → V_pre 大但 output 还是小 → W_out 为了把小信号放成 logits 不得不一直长 → 反向梯度链 `(∂loss/∂out) · output · activation^T` 在 bf16 下溢出 → NaN。

**确诊**：quantal 下 Muon LR 0.02 太大。supra 不爆是因为 supra 的输出对 V_pre 弹性吸收 W_in 增长。**修复路径**：
1. 降 muon_lr 到 0.005（4x）—— 简单一步实验。
2. 或矩阵改 Adam（保留 Lion 给神经元参数）—— quantal-only 路径。
3. 加 weight decay 给 Muon。
4. 加 post-spike normalization 让 quantal output 也能自适应（架构改动，最后选项）。

⚠️ **重要修正**（之前误报「quantal+MAL 最优」）：那个结论是把 quantal@muon_lr=0.005 跟 supra@muon_lr=0.02 比的，**不是 apples-to-apples**。补了 supra/supra+AHP @ muon_lr=0.005 后，在同 LR 下：

**这一轮的结论（修正版）**：
1. **MAL > Adam 全面，且 muon_lr=0.005 > muon_lr=0.02**：

   - MAL muon_lr=0.02: supra 5.01 / quantal NaN (需 0.005)

2. **同 LR (0.005) 下，supra 略胜 quantal**：4.25-4.27 vs 4.38（~0.12 nat），且 **supra 的 hidden 发放率高很多**（μ0.22 vs μ0.07）。之前「quantal 最优」是 LR 不对齐造成的假象。

→ **推荐配置（当前小规模证据，待大规模验证）**：
   - 优化器：**Muon(matrices) + Adam(embed/norm) + Lion(neuron scalars)，muon_lr=0.005, lion_lr=1e-4, clamp v_th/ahp≥1e-4** ← 这是确凿的 win
   - spike_mode：supra（loss 略好、发放率高）或 quantal（膜保留更多历史、K 步更强）—— **在 1500 步小规模上没决出胜负，留给长训练**。如果重视「膜承载历史上下文 + K 步递归 load-bearing」这两个设计目标，选 quantal；如果纯看小规模 loss，选 supra。
   - AHP：可有可无，默认 False（少一个旋钮）。
   - **本轮只 1500 步 / 300 万 token，是「能不能学起来 / 哪个优化器」级别的结论，不是「哪个架构更强」的最终结论**。

## §3.8 quantal NaN 根因调查 + 稳化（2026-05-12，决定性）

§3.4 那批"安全配置"（muon_lr=0.003/0.005 + muon_wd + grad_clip + NaN-skip）只是把 NaN 从 step~115 拖到 step~1000，没根治（§3.7 的 1.24B quantal 仍在 step964 NaN）。这一节做了系统诊断 + 找到根因 + 稳化。

### 诊断方法（`scripts/v4_diag_nan.py`，加了 anomaly mode + Vpost_max trace + freeze 开关）
- `--anomaly`（`torch.autograd.set_detect_anomaly`）精确定位：**NaN 来自 `_SegmentedPLIFSelectiveBackward` 的第 0 个输出 = `grad_beta`**（SNNBlock hidden 神经元 segmented 递归对 per-position β 的梯度）。

### 中间错误诊断（保留作记录 —— 这些是对路上的错误推断）
1. **「β→1 (bf16 round) → 膜成无界积分器」**（commit `a291f2e` 的 β-cap @0.99）—— 部分对（β=sigmoid(W_beta_x(x)) 确实在 bf16 下对高 β 通道 round 成 1.0），但 **不是 NaN 的直接原因**：β-cap 把膜从 ~6e6 压到 ~6e4 后**还是 NaN**（diag muon_lr=0.02 → NaN@88）。5e4 远低于溢出（3.4e38），所以膜量级不是溢出源。
2. **「Muon 推矩阵 → 膜随 u 无界涨 → backward grad_β∝V 溢出」**—— 量级估算不对（膜 ~6e4，grad_β ~ 6144·g_out·6e4 ~ 1e7，finite，不溢出）。

### 真根因（决定性 —— `v4_diag_nan.py` 加 backward hook 直接测量验证）
backward hook 捕获 `segmented_plif_selective` 的输入/输出梯度：
```
sel-call#0 (layer 0):  gout = 2.65e-4 (finite, ∂L/∂SNNBlock输出, 从下游 W_out 传回)
                       →  gbeta = nan(INF!)  gu = inf(INF!)  gvth = inf(INF!)
```
**kernel 拿到一个 finite 的 ~2.65e-4 输入梯度, 内部放大成 inf（>1e38× 放大）。** 这是 **BPTT 递归 Jacobian 不稳定**：

膜递归 `V_pre[t] = β[t]·V_post[t-1] + u[t]`、`V_post[t] = V_pre[t] − (v_th[t]+ahp[t])·s[t]`，quantal STE 下 `∂s/∂V_pre = g_surr = α·σ·(1−σ)`，峰值 `g_surr_max = α/4`（α = surrogate_alpha = 4）。backward 一步乘子：
$$J_t = \frac{\partial V_{post,t}}{\partial V_{pre,t}}\cdot\frac{\partial V_{pre,t}}{\partial V_{post,t-1}} = \big(1 - (v_{th,t}+ahp_t)\cdot g_{surr,t}\big)\cdot\beta_t$$
`J_t` 上界 `= β < 1`（永远 fine）；下界 `= β·(1 − (v_th+ahp)·α/4)`。**当 `(v_th+ahp)·α/4 > 1+1/β`（≈2，β≈0.99）时 `|J_t| > 1` → 跨多个 transition 帧（V_pre 在阈值附近、g_surr 取峰值）`|J|^n` 指数发散 → grad 溢出 → NaN。**

**触发**：SelectivePLIF hidden 的有效阈值 `v_th_hidden = v_th_min + |W_th_x(x)|` —— `W_th_x` 被 Muon 推大（α=4 下门槛是 `v_th_hidden > ~1-2`）。**更深一层**：quantal `out = v_th·s` 把「输出幅度」绑成「阈值」→ loss 要一个非微小的 SNNBlock 输出 → 驱动 `v_th_hidden`（即 `|W_th_x(x)|`）涨大。Muon 再帮推一把。

**实测轨迹（diag muon_lr=0.02，新码 bigvth+bias重构+β-cap，旧 fixed-α）：**
| step | v_th_hidden_max | \|J\|_max（=β·max(1,\|1−v_th·α/4\|)）| W_th_x_max | Vpost_max | 状态 |
|---|---|---|---|---|---|
| 0 | 0.43 | 0.988 | 0.04 | 5.0 | 稳 |
| 10 | 1.44 | 0.988 | 0.44 | 46 | 稳（\|1−1.44\|<1）|
| 20 | 4.47 | **3.10** | 1.09 | 287 | ❌ \|J\|>1，递归发散 |
| 50 | 10.9 | **9.02** | 4.22 | 4e4 | |
| ~88 | — | — | — | — | **NaN（layer-0 SNNBlock backward）** |

**supra 为什么不爆**：`V_post = min(V_pre, v_th)` → `∂V_post/∂V_pre = 𝟙[V_pre<v_th] ∈ {0,1}` → `J = β·{0,1} ∈ [0,β]` 恒 contractive，与 v_th、电流无关（exact ReLU 梯度，没有 surrogate）。代价：膜被钉死、不携带历史（v3 旧设计）。

### 修法：adaptive surrogate sharpness（commit `6226176`）
让 surrogate 的"涂抹宽度"自适应：`α_eff = min(α_base, 4/(v_th+ahp))`。则 `(v_th+ahp)·α_eff/4 ≤ 1` → `∂V_post/∂V_pre = 1−(v_th+ahp)·g_surr ∈ [0,1]` → `J ∈ [0,β]` → `|J| ≤ β < 1` **恒 contractive，不论 v_th_hidden 多大**。
- `v_th+ahp` 小时 `α_eff = α_base`（不变）；`v_th` 被驱动大时 `α_eff` 反比缩小、surrogate bump 加宽（积分仍 =1，仍是 δ 的合法近似）—— 不把 BPTT 推到发散区。
- **不改 forward**（forward 用 hard step `𝟙[V_pre>v_th]`，与 α 无关）；**不 clamp v_th、不重参数化、不动 Muon/优化器**。`.detach()` 不 backprop 穿过 `α_eff` 的自适应（kernel 用 hand-written grad，与 pytorch ref 对齐）。
- 跟 codebase 已有的 "MPD-AGL adaptive surrogate（α ∝ 1/v_th）" 同源 —— segmented PLIF 的 backward kernel 之前没用上、用的是固定 `ALPHA=4`。
- 改了：`_segmented_plif_bwd_{rowparam,selective}_kernel`（Triton）+ `_spike_step`（pytorch ref）。对 rowparam 神经元（v_th~0.14-0.41 < 1）是 no-op。`test_segmented_plif` + `test_v4_end_to_end` 14/14 pass。

### 验证（adaptive-α）
- diag muon_lr=0.02, max200（旧码这配置 NaN@88）：用 adaptive-α 后 **DIAG END at step 200，无 NaN**。`|J|_max` 全程恒 = **0.988281**（= β，contractive 上界）—— 即使 `v_th_hidden_max` 涨到 **39**（W_th_x 被 Muon 从 0.04 推到 5.6），`|J|` 纹丝不动。`Vpost_max`（forward 膜）随 u 涨到 ~3.7e4（adaptive-α 不动 forward），但远低于溢出。loss 正常降（3.6-6.8 抖）。对照：旧码（fixed α=4）同 step 50 时 vthH=10.9 → |J|=9.02 → NaN@88。**数学推导（J∈[0,β]）与实测分毫不差。**
- 干净 quantal 3k derisk（adaptive-α 新码 = β-cap+bias重构+bigvth+adaptive-α，muon_lr=0.005+muon_wd 0.01+grad_clip 0.5+lsuv）：**DONE 3000 步，0 NaN-skip**，final_loss(last100) = 4.4285（`derisk_q5_3k.pt`）。对照同配置但**无 adaptive-α** 的 derisk_q4：**1 NaN-skip**，final_loss 4.4830。→ **adaptive-α：0 skip vs 1 skip，loss 还略好**。**quantal 稳化在 derisk 规模确认完成。**

- 顺便给 `v4_neuron_ablation.py` 加了 `--save_every`（默认 500，周期 ckpt 防 run 被杀全丢）。

### 全否定的「修法」（保留作记录）
bigvth init（quantal PLIFNode.v_th uniform(0.15,0.40)，commit `f7ad0fa`）/ β-cap @0.99（`a291f2e`）/ muon_wd 加大 / vth_reg（朝-init 二次正则，实测有害）/ AHP（ahp 大反引入新不稳）/ surrogate_alpha 直接调小（v_th 无界 → 没固定 α 够用）—— **都不是 quantal NaN 的修法**（顶多延迟）。只有 adaptive-α 是根治。注：bigvth init / bias 重构 / β-cap 这几个改动**保留**（它们各自有别的好处：bigvth 防 input 神经元输出微小、bias 重构是 Llama 式无偏置、β-cap 防 bf16 β=1.0），只是它们不是 NaN 的根治。

## §3.9 bias 重构：删 SNNBlock 的 b_beta/b_alpha/b_th（Llama 式无偏置，2026-05-12）

commit `e36bb54` + `f7ad0fa` + `cbad214`。用户定调：现代 LLM 无偏置项（Linear `bias=False` —— 这点代码本来就满足；`b_beta`/`b_alpha`/`b_th` 是写成 `W_*_x(x) + b_*` 的偏置，等价于 `bias=True`，要删）。

### 改了什么
- 删 `SNNBlock.b_beta` / `b_alpha` / `b_th`（各 shape `(DN,)`，每 SNNBlock 层 3·DN 个参数，D=1024/N=8 → 8192 ×3 × ~18 层 ≈ 44 万参数）。选择性方程变：`β = sigmoid(W_beta_x(x))` / `α = softplus(W_alpha_x(x))` / `v_th = v_th_min + |W_th_x(x)|`。
- per-channel timescale/threshold 的 **init 多样性挪进 `W_*_x` 的行偏移**：给输出通道 i 的那一行加常数偏移 ≈ `原 b_*目标_i / (D·dc_est)`。原理：SNNBlock 输入 `v_in`（input 神经元脉冲输出）非负、有正 DC ≈ dc_est → `W_*_x(v_in)[i] ≈ 行均值_i·Σ_j v_in_j ≈ 行均值_i·D·dc_est`，令其 = 目标 → 行偏移 = 目标/(D·dc_est)。`dc_est` mode-aware（quantal 0.08 / supra 0.15，粗估）。训练中随机选择性分量长起来后这个偏移会被洗掉（符合预期 —— "bias 藏进 weight init"）。
- `compensate_modulation_gradients` 改 no-op（b_* 已删；W_*_x 走 Muon，更新幅度不依赖梯度幅度，不需饱和补偿）。清理 `get_param_groups` / `_NEURON_PARAM_SUFFIXES`（含把 `.ahp` 加进 fp32 列表）/ `utils/param_groups.py` / `utils/dashboard.py`（b_* 监控加 hasattr 守卫）/ `scripts/v4_memory_analysis.py`（E1 β 谱用 W_beta_x 行均值代理）。
- 表达性影响：核心表达性（逐通道神经元内禀参数 .w/.v_th/.ahp 满逐通道 1D + β/α/v_th 对 D 维输入的满秩选择性）**不变**；唯一削减是那 3 个专门的逐通道偏置参数（用户要的），其 init 角色 fold 进 W_*_x 行（训练中 W_*_x 行结构可近似充当 bias）。
- smoke + 测试：tiny CPU 模型两模式 forward loss finite，β_init ≈ [0.75, 0.99]；`test_segmented_plif.py` + `test_v4_end_to_end.py` 14/14 通过。

### 混合精度 + 推理 state 实测（`scripts/v4_check_dtype.py`，D=1024/N=8/24L/D_ff=3072/vocab=128387）
- **1.236B 参数 ≈ 2.47 GB（全 bf16，含 .w/.v_th/.ahp，0 fp32 param —— 见 §3.10 stochastic rounding）**。`b_*` 参数 `[]`、Linear `bias=True` `[]` —— 确认。（早期版本 .w/.v_th 保 fp32 = 1.6 MB，现在 bf16 ~0.8 MB，可忽略。）
- **推理常驻 state（batch=1）≈ 1.378 MB**（121 个 state tensor：每 SNNBlock 层 `hidden_neuron.v (1,8192) fp32`、`gate/up_neuron.v (1,3072)`、input 神经元 `(1,1024)`；每 memory 层 SNNAttention `M_state (1, D_key, D_value)`、`pos_offset` 标量）—— **携带状态固定大小、不随上下文长度增长**（线性注意力，无 KV cache）。
  - ⚠️ **但 `forward_parallel(h)` 本身的 activation 是 O(len(h))**：`kv_outer/kv_gated/M_all` 都是 `(T, batch, D_key, D_value)`，FFN 子层 `(T·K, batch, D)`。所以**按整条上下文一次跑 → 显存 O(T)**；要恒定显存须**分块跑**（每次 `forward_parallel(chunk)`，靠 `M_state`/`pos_offset` 续传，峰值 O(chunk)）—— 架构支持，分块推理 driver 待确认/补。

## §3.10 全 bf16（含神经元参数）+ stochastic rounding（2026-05-12）

之前神经元参数 `.w/.v_th/.ahp` 保 fp32（`promote_neuron_params_fp32`），原因是 bf16 下小更新被量化掉、得不到训练。验证 + 解决：

### 问题验证（`scripts/v4_test_bf16_neurons.py`，3 变体各 1000 步）
| 变体 | `.v_th` 相对 init 变化（min/med/max）| `.w` | loss Δ |
|---|---|---|---|
| full-bf16 + MAL（**无** SR）| **0.00 / 0.00 / 0.00（完全没训）** | 3.7e-3~2.3e-2（有训）| +0.025 |
| full-bf16 + MAL（**有** SR）| **0.138 / 0.187 / 0.281（训得动了）** | 0.089~0.153 | +0.025 |
| mixed (fp32 神经元) + MAL [旧默认] | 0.105 / 0.176 / 0.282 | 0.065~0.165 | +0.024 |

→ 根因：bf16 在 v_th 典型值（~0.1-0.4）附近的精度 ~0.001-0.002，Lion 的 `lion_lr=1e-4` 步长比它小一个量级 → round-to-nearest 每步把更新吃掉；且 v_th 梯度符号来回摆 → cumulative 也凑不够跨一个 bf16 格子。`.w` 碰巧能动（接近 0 的元素 bf16 精度更细 + 梯度更一致）。

### 解决：stochastic rounding（commit `f70993e`）
- `utils/muon_adam_lion.py`：加 `_stochastic_round_to_bf16`（int-bit trick：fp32 看成 int32、低 16 bit 加 `[0,2^16)` 随机数、再截断到高 16 bit → 残差 r 以概率 r/2^16 进位 → 无偏）+ `_apply_step(p, update, lr)`（p 是 bf16 时在 fp32 里算 `p − lr·update` 再 SR 回 bf16；fp32 param 是 no-op）；Muon/Adam/Lion 三条 path 都走 `_apply_step`；Lion 的 momentum buffer 改 fp32（防 EMA 累积小梯度时丢精度）。
- 这是 bf16/fp8 低精度训练的**标准技术**，不增加 per-param fp32 状态，可证明无偏（E[结果]=精确值），对所有 bf16 param 一律适用（矩阵也一样，SR ≥ round-to-nearest，无副作用 → 默认开）。
- 不那么优雅的替代（弃）：调大 lion_lr/neuron_lr_mult（band-aid，要重验）；Kahan/error-feedback（额外状态 ≈ fp32 大小）；log 空间重参数化（不解决 —— 浮点精度是相对的，固定绝对更新不论值多大都 sub-ULP）。
- `promote_neuron_params_fp32` 现在只 promote k_predictor 的 EMA buffer 到 fp32（gradient-free EMA 累积小分数、不经优化器的 SR；且极小）。`from_pretrained` / `v4_neuron_ablation` / `v4_diag_nan` / `v4_check_dtype` 的 cast 改成全 bf16。

### 验证
- `v4_check_dtype`（1.24B 配置）：**0 fp32 param**（input_neuron.w/.v_th 现在都 bf16），全模型 2.47 GB bf16。
- `test_segmented_plif` + `test_v4_end_to_end` 14/14 pass。
- **all-bf16 稳定性 redo（quantal/supra 5k derisk，全 bf16 + adaptive-α + SR + 推荐配置）—— DONE：**
  - **quantal（`derisk_q6_bf16_5k.pt`）：5000 步，0 NaN-skip，final_loss(last100) = 3.8547，ms_step 1407。**
  - supra（`derisk_s6_bf16_5k.pt`）：5000 步，0 NaN-skip，final 3.7054，ms_step 1446。（supra 比 quantal 略低 ~0.15 nat，跟 §3.6/§3.7 一致。）
  - vs fp32-神经元的 derisk_q5（3k 步）final 4.43 —— q6 跑了 5000 步（更多训练→loss 更低）且 0 skip → **bf16 神经元参数训得动（stochastic rounding 在真训练里生效）、quantal 在全 bf16 下稳定**。「quantal 稳化（adaptive-α）+ bf16 神经元参数（SR）」两件都确认完成。`derisk_q6_bf16_5k.pt` = canonical ckpt（Track A 记忆归因用）。

## §4a quantal 保住 K 步动力学（重要观察 — 副带解决「PonderNet 退化 K」）

实测对比（MAL 优化器, 1500 步）：
- **supra**: E[K]_out 从 6.5 (init) → step 1000 **3.35** → step 1400 **2.60**（PonderNet 学到只跑 ~2-3 帧）
- **quantal / quantal+AHP**: E[K]_out 一直稳在 **~7.6-8.1**（PonderNet 学到值得跑 ~8 帧）

**机制**：
- supra 的 `output = relu(V_pre - v_th)` + 软重置到 v_th → 发放就把超阈部分掏空、膜钉回阈值 → **膜几帧就 settle（收敛到稳态）** → 多跑帧没额外信息 → PonderNet 学着提前 halt → **K 退化到 ~1，K 步串行 SNN 动力学被实质废掉**（架构核心卖点失效）
- quantal 的 `V_post = V_pre - v_th·s - ahp·s`，**超阈余量留膜里** → 膜跨帧持续积分、攒到够整数份 v_th 就 burst（rate code）→ **每多跑一帧都是真计算** → PonderNet 学到「这值得跑更多帧」→ E[K] 稳在高位 → **K 步递归动力学保住**

→ quantal 一个改动同时：① 膜保留历史上下文（原始设计动机，§1）② 让多步递归「load-bearing」从而 PonderNet 不会把 K 退化掉（本来要单独解决的问题）。**这是 quantal vs supra 的一个独立优势，且不依赖小训练规模**（机制是结构性的）。

注：副作用 —— quantal 推理时 early-stop 省的 FLOP 更少（E[K]≈8 vs supra E[K]≈2.6）。但 supra 的 E[K]≈2.6 是「把 SNN 跑废了」换来的，不算真省。trade-off：quantal 保留 SNN 计算能力但推理慢一些。

## §4 模型尺寸效应（重要观察）
小模型（D=384 / 8 层）quantal 原样训练正常（hidden 0.22→0.32）；大模型（D=512 / 12 层）quantal hidden 起步就 0.006、塌到 0。

**原因（理论）**：quantal output = v_th·s ≈ 0.02; W_in (D=512) → DN=8192, kaiming `~N(0, 2/D)`. (W_in @ output) per-channel std = sqrt(2·E[output²]) = sqrt(2·0.02²·0.49) ≈ 0.02. 即 hidden 神经元的 V_pre 期望幅度 ~0.02，hidden v_th ≈ v_th_min + |...| ≈ 0.02-0.05。**V_pre ~ v_th 量级 → 发放率随机** → 略低于阈值时几乎不发放。

更深的层级联衰减：input_neuron 输出 ~0.02 → W_in → hidden V_pre ~0.02 → hidden 输出 ~0.02·firing_rate ≈ 0.006 → 下一层 W_in → ~0.006 → ...

→ quantal 在大模型下需要**初始化解决级联问题**（论文核心 contribution 之一）。

## §4b 记忆归因：历史信息在 SNNBlock vs SNNAttention 之间如何分配（`scripts/v4_memory_analysis.py`）

**目标**：数学严谨地验证作者的推测 —— SNNAttention（类海马体线性脉冲注意力）= 长程记忆；SNNBlock（SelectivePLIF SSM-state 类比）= 短/近期记忆；SNNBlock 内部本身也有长短期分化。

### E1 β 时间常数谱（design-level，定义：膜递推 ∂V_t/∂V_{t-1}=β → 影响按 β^Δ 衰减 → 记忆时间常数 τ = -1/ln β）
在 290M 配置模型上（fresh-init，因为 β 的 base 来自初始化设计）：

| 组件 | β 来源 | τ 范围（单位 = K-frame 步）| 桶分布 [<3, 3-10, 10-30, 30-100, >100] | 记忆性质 |
|---|---|---|---|---|
| input_neuron1/2, gate_neuron, up_neuron, output_neuron (PLIFNode) | sigmoid(w), w~N(0,0.5²) | τ ≈ 0.7-3.9（中位 ~1.4）| ~97% <3 | **快 / 近期专属**（~1-2 步）|
| SNNBlock hidden_neuron (SelectivePLIF) | β_base=sigmoid(b_beta), b_beta init = logit linspace(0.80,0.99) + N(0,0.1) 扰动 | τ_base ≈ 4-110（中位 ~9）| ~0% <3, ~55% [3-10], ~31% [10-30], ~11% [30-100], ~3% >100 | **多时间尺度**（4 到 110）—— **SNNBlock 内部就有长短期分化** |

SNNAttention：线性注意力 + RoPE → 位置 s 对位置 t 输出的有效权重 ∝ Σ_j cos((t-s)·θ_j)，θ_j = base^(-2j/d_head)，base=10000, d_head=128（=D_key）。「记忆波长」λ_j = 2π/θ_j：

| | 最短 λ（最高频）| 中位（25/50/75%）| 最长 λ（最低频）|
|---|---|---|---|
| SNNAttention RoPE | 6.3 tokens | 60.7 / 586 / 5652 | **54410 tokens** |

→ **设计层面结论（支持作者推测）**：
- PLIFNode 神经元：~1-2 步 → 纯近期
- SNNBlock hidden_neuron：~4-110 K-frame 步（≈ token，因 carry 是 per-token）→ 短到中程，**内部多时间尺度**
- SNNAttention：~6 到 ~54000 tokens → 短到**极长程**
- **SNNBlock 负责短/近期，SNNAttention 是长程记忆的载体**；且 SNNBlock 自身有长短期分化（b_beta linspace [0.80,0.99]）。

### E2 经验状态自相关（定义：ρ(Δ) = mean_c pearson_t(state_t[c], state_{t-Δ}[c])；衰减到 1/e 的 Δ = 经验记忆时间常数）
在 fresh-init 模型上、val 序列（seq 256）—— 抓 SNNBlock hidden_neuron 跨 token 携带的膜电位 V_post[k_t]：
- 各层 ρ(Δ): Δ1≈0.52-0.63, Δ2≈0.30-0.39, Δ4≈0.12-0.19, Δ8≈0.03-0.10, Δ16≈0-0.06, Δ≥32≈0
- **经验 τ_emp(1/e) ≈ 1.7-2.3 tokens**

→ 经验上 SNNBlock 携带状态只自相关 ~2 tokens —— 远小于 β_base 的中位 ~9（更别说 ~110 的尾巴）。原因：(a) PLIFNode 们 β≈0.5 → τ≈1.4 → 这些「快」神经元主导了 input_neuron 链；(b) fresh-init 时膜被当前 token 的新输入主导，递归历史项小（β^K 内衰减）；(c) 只有 hidden_neuron 的 ~3% 慢通道（β≈0.99）携带 ~100-token 记忆，混入均值后被快通道淹没。**训练后再测会更有意义**（模型可能学会用慢通道）。

### E3 因果扰动影响（定义：扰动 emb at pos T-1-Δ → 末位 logits 变化 ‖Δlogits_T‖/‖ε‖ = influence(Δ)；component 归因：关掉 SNNAttention 跨位置混合后 influence(Δ) 在长 Δ 塌掉 ⇒ 长程靠 SNNAttention）
- **当前状态：有 bug，结果不可信** —— fresh-init 上 influence(Δ) 全是噪声幅度（~20）无衰减趋势，no-attn-mix 的 ratio 也接近 1。怀疑全局 `torch.cumsum` monkeypatch 在大模型（有 `checkpoint` 包裹）下没可靠传到 SNNAttention 层。待：① 在 trained ckpt 上复测；② 换更直接的 ablation（直接 override `SNNAttentionDecoderLayer.forward_parallel` 跳过 attn 子层）。
- **架构论证（不依赖实验，结构性事实）**：模型里跨位置信息路径只有三条：(1) SNNAttention 的无衰减 cumsum（RoPE 调制）—— 跨度 6 到 ~54000 tokens；(2) SNNBlock 的 segmented-PLIF 跨 token carry —— 快通道 ~1-2 tokens，慢通道（~3%）~100 tokens；(3) SNNDecoderLayer 的皮层侧向连接 —— 1 token 回看。**所以任何超过 ~100 tokens 的依赖必然走 SNNAttention**；~10-100 tokens 范围 SNNBlock 的慢通道和 SNNAttention 都能做；~1-10 tokens 范围三者都能。

### 在 trained ckpt 上的复测（`/workspace/quantal_mal_trained.pt`，quantal+MAL, 1500 步, final_loss 4.378）

**E1（trained 后的 β 谱）—— 多时间尺度结构被保住了**：
- PLIFNode output_neuron: τ 还是 ~1-2（98% <3）—— 训练没改变，仍是快/近期神经元
- SNNBlock hidden_neuron β_base: τ_base 4-115, 桶 ~55% [3-10], ~31% [10-30], ~10-11% [30-100], ~3-4% >100, β_base ∈ [0.72, 0.994] —— **跟初始化几乎一样**（Lion 的小步更新 ×1500 步 + 多时间尺度初始化很 sticky）→ 训练保住了 SNNBlock 内部的长短期分化，没塌成单峰。

**E2（trained 后的经验状态自相关）—— 出现了「层级记忆梯度」**：
| 层 | τ_emp(1/e) | ρ(Δ=4) | ρ(Δ=16) | ρ(Δ=64) |

| layer 0 | 2.7 | 0.27 | 0.08 | 0.02 |

| layer 9 | 6.5 | 0.45 | 0.18 | 0.05 |

- fresh-init 时所有层 τ_emp ≈ 2；**训练后早层（0-2）≈ 2.5-3.8，中层（4-6）≈ 8-11，晚层（8-10）≈ 6.5-7.8** —— **模型学会了在中层用更长的递归记忆**（中层 ~10 tokens，早层 ~3 tokens）。
- 所有层 ρ(Δ) 在 Δ≈64 还微正（~0.03-0.05），Δ≥128 转负（噪声）→ SNNBlock 的携带膜电位经验上自相关 ~32-64 tokens（快成分 ~3-10 tokens + 慢尾 ~3% 通道延到 ~64 tokens），跟 bimodal β 一致。
- → **SNNBlock 内部既有「层内多时间尺度」（β bimodal）又有「跨层梯度」（早层短、中层长）的双重分化**。这是 SNNBlock 负责短/中程记忆的实锤；最长 ~64 tokens。**超过 ~64-100 tokens 的依赖必须走 SNNAttention**（RoPE 波长 6→54000 tokens）。

**E3（因果扰动，16-ε 平均后）**：

|---|---|---|
| 1 | 99.5 | 211 |
| 2 | 77.4 | 126 |
| 4 | 92.3 | 139 |
| 8 | 83.1 | 109 |
| 16 | 76.2 | 100 |
| 32 | 75.8 | 111 |
| 64 | 75.4 | 111 |
| 128 | 69.7 | 110 |
| 256 | 72.9 | 84 |

- **full 模型的 influence(Δ) 大致平坦（Δ=1→256 从 ~100 微降到 ~73，~25% 衰减）** —— 扰动 256 tokens 前的 embedding 对末位 logits 的影响 ≈ 扰动 1 token 前的。**这就是 SNNAttention 的无衰减 cumsum + 低频 RoPE 在干活** —— 给整模型一个 transformer-like 的平坦长程注意力（纯 SSM/SNN 会指数衰减；这里不衰减）。
- **no-attn-mix 的 ratio 不可靠**：disable cumsum 后 per-position 注意力 ≠ 训练时的 cumsum 注意力 → 模型进入 OOD 状态 → 对任何扰动都更敏感（influence 整体抬高到 ~100-210）→ ratio 常 >1，不能解读成「SNNAttention 不重要」。要干净地做 component 归因，得在 OOD 不严重的情况下（比如训练时就用 windowed cumsum，或在 trained-with-ablation 的模型上比）。**留待后续**。
- 但 **full 的平坦 influence + E2 的 SNNBlock τ_emp ~3-11 tokens（~64 tokens 死透）合起来就够了**：SNNBlock 携带的递归记忆 ~64 tokens 内有用，超过就死；full 模型 ≥256 tokens 还平坦 → 那 ~64-256+ 这段必然是 SNNAttention 扛的（架构上也只有它能做）。**SNNBlock = 短/中程（≤64 tokens, 有层内 + 跨层多时间尺度），SNNAttention = 平坦长程载体。作者的推测成立。**

### P2 ablation 结果（{supra,quantal}×{LSUV,no-LSUV} @ MAL muon_lr=0.005，1500 步全完成）

| 配置 | no-LSUV final_loss / hidden μ / gate0 | LSUV final_loss / hidden μ / gate0 |
|---|---|---|
| supra | 4.267 / 0.221 / 0.27 | 4.240 / 0.227 / — |
| quantal | 4.378 / 0.065（SNN 死）/ 0.26 | 4.450 / 0.189（SNN 活）/ 0.30 |

**结论**：
- **supra**：LSUV 基本不动 loss（4.27→4.24），hidden 本来就没塌（~0.22）。
- **quantal**：LSUV 把 SNN 救活（hidden 0.065→0.189）但 **loss 反而略差**（4.378→4.450, +0.07）。即在 1500 步小规模上，「活的 quantal SNN」比「死的 quantal SNN」loss 略差，两者都比 supra（4.24-4.27）略差 ~0.1-0.2 nat。
- 解读：活的 SNN 块是个更复杂、更难训的组件，1500 步还在找好权重 → 这个规模上净贡献 ≈ 0 到略负。**赌的是长训练 / 真任务上活的 SNN 才发力** —— 所以前期的 recall probe / 时间序列 demo 就是要验证「活的 SNN 到底有没有在做有用的计算」。LSUV 对 quantal **是必需的**（不然 SNN 死，架构设计意图落空），只是 1500 步看不出收益。
- → **supra vs quantal 在 1500 步规模没决出胜负（supra 略前），架构选择留给前期的 probe + 真正的长训练**。优化器 MAL(muon_lr=0.005) + LSUV 是确凿的配置。
